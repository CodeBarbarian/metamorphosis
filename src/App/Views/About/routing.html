Get started with Routing for Metamorphosis

Routing is responsible for the way people navigate the application. There are so many ways routing can be done, and many pitfalls. Routing may be your friend if you use it correctly

The standard route: $Router->add('{controller}/{action}'); covers the majority of uses.

Using routing with parameters

Let us say you are accessing a specific item https://websitename.org/database/items/53. Then you need to add the following route into your routing table:

$Router->add('database/items/{id:[\da-f]+}', ['controller' => 'Database', 'action' => 'items']);

When having characters in the address it is good practice to only allow hexadecimal input.

You can use almost any custom regex you want in the parameters.

Using routing with custom URIs

To create custom routes. Just add them to your routing table and select a controller and an action

            $Router->add('test/test1/test2/test3/test4', ['controller' => 'About', 'action' => 'index']);
