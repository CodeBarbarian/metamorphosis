Get started with Configurations for Metamorphosis

Configuration files are an easy way to store hard coded data for your controllers, models, plugins and other objects.

Using Configurations

To access any configuration value, access it using static method: \App\Config\CONFIGFILE::PROPERTY

In this example we access the static property of the Error.php for showing errors or not: 

            if (\App\Config\Error::SHOW_ERRORS)

Creating Configurations

To create a configuration file, place it in the Configurations directory and make sure to set the correct namespace

As an example we use the Error Configuration file: 

            namespace App\Config;

            use Core\Config;

            /**
             * Error Config
             * @version: PHP: 8.1
             *
             * @Error
             */
            class Error extends Config {
                /**
                 * Show or hide error messages on screen
                 * @var boolean
                 */
                const SHOW_ERRORS = true;
            }

Production vs Development Environment

Development Environment

When developing applications, it is recommended to use the following settings on in your configurations:

  • In ./App/Config/Error.php: SHOW_ERRORS = true;
  • In ./App/Config/Site.php: SECURE_SERVER_COMMUNICATION = false
  • In ./App/Config/Site.php: SYSTEM_ENVIRONMENT = 'development';
  • In ./App/Config/ControllerConfig/Framework.php: ALLOW_FRAMEWORK_SETTINGS = true

Production Environment

When you have finished developing your application, it is recommended to use the following settings in your configurations:

  • In ./App/Config/Error.php: SHOW_ERRORS = false;
  • In ./App/Config/Site.php: SECURE_SERVER_COMMUNICATION = true
  • In ./App/Config/Site.php: SYSTEM_ENVIRONMENT = 'production';
  • In ./App/Config/ControllerConfig/Framework.php: ALLOW_FRAMEWORK_SETTINGS = false
